Minecraft Server Deployment on AWS

Cloud Infrastructure Project | Spring 2023

Project Overview

Designed, deployed, and managed a scalable Minecraft gaming server on Amazon Web Services cloud infrastructure. The project focused on creating a reliable, cost-effective multiplayer gaming environment while learning cloud computing principles and system administration.

The server supported up to 24 concurrent players with optimized performance and implemented automated systems for backups, monitoring, and cost control.

Technical Implementation

The server was built using the following AWS services and technologies:

  • AWS EC2: t3.xlarge instance (4 vCPUs, 16GB RAM) running Ubuntu Server 22.04
  • AWS S3: Automated daily world backups with 30-day retention
  • AWS Route 53: Custom DNS (minecraft.alexchen.tech) for easy server access
  • AWS CloudWatch: Performance monitoring and alerting
  • PaperMC: High-performance Minecraft server software
  • Bash Scripting: Automation for updates, backups, and maintenance

Key Features Implemented:

  • Automated daily backups to S3 with versioning enabled
  • Scheduled instance shutdown during off-peak hours (12 AM - 4 PM) to reduce costs by 65%
  • Custom performance monitoring dashboard using CloudWatch metrics
  • Security hardening with minimal open ports (only 25565 for Minecraft)
  • Player whitelist system for enhanced server security
  • Dynamic resource monitoring with auto-alerts for high usage

Challenges & Solutions

High Latency for Distant Players
Implemented AWS CloudFront distribution for static assets and optimized network routing, reducing latency from 120ms to 85ms for cross-country players.
Cost Management
Created Lambda functions to automatically stop the instance during low-usage hours and implemented budget alerts, maintaining monthly costs under $45.
World Data Corruption Risk
Designed a robust backup system with daily snapshots and weekly full backups stored across multiple AWS regions for disaster recovery.
Performance Optimization
Tuned JVM arguments and implemented chunk pre-loading to reduce lag during peak player activity.
