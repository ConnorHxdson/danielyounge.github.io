Checkerboard Game Simulator with AI

Object-Oriented Programming & AI Project | Fall 2023

Project Overview

Developed a complete Checkers game simulator with graphical interface, rule enforcement, and multiple AI opponent difficulty levels using Java and JavaFX. The project focused on implementing game logic, artificial intelligence algorithms, and object-oriented design principles.

The simulator supports multiple game modes including Player vs Player, Player vs AI, and AI vs AI simulation with statistical analysis. It features a fully animated interface, move validation, and comprehensive game state management.

Technical Implementation

Core Architecture:

  • Programming Language: Java 17 with JavaFX for GUI
  • Design Patterns: Model-View-Controller (MVC), Singleton, Strategy
  • Game Engine: Custom-built checkers rule engine
  • AI System: Minimax algorithm with alpha-beta pruning
  • Data Persistence: JSON file storage for game states and statistics

Key Components:

  • Board Class: 8x8 grid representation with piece management
  • Piece Class Hierarchy: Abstract Piece class with Man and King subclasses
  • Game Controller: Central game logic and state management
  • Move Validator: Comprehensive rule checking including mandatory jumps
  • AI Engine: Three difficulty levels with adjustable search depth
  • UI Renderer: Interactive graphical interface with animations
  • Statistics Module: Game analytics and performance tracking

  • Complete Rule Implementation: Standard American Checkers rules with forced captures
  • Multiple Game Modes:

AI Algorithm Implementation

Minimax with Alpha-Beta Pruning:

function alphabeta(node, depth, α, β, maximizingPlayer) is
    if depth = 0 or node is terminal then
        return evaluate(node)
    if maximizingPlayer then
        value := -∞
        for each child of node do
            value := max(value, alphabeta(child, depth-1, α, β, FALSE))
            α := max(α, value)
            if α ≥ β then
                break // β cutoff
        return value
    else
        value := +∞
        for each child of node do
            value := min(value, alphabeta(child, depth-1, α, β, TRUE))
            β := min(β, value)
            if β ≤ α then
                break // α cutoff
        return value

Board Evaluation Function:

The AI uses a weighted evaluation function considering:

  • Material advantage (piece count)
  • King value multiplier (kings worth 1.5× regular pieces)
  • Positional advantage (center control, back row safety)
  • Mobility (number of legal moves available)
  • Forced capture opportunities

Performance Optimization:

  • Move ordering heuristics for better alpha-beta pruning
  • Transposition table for caching board evaluations
  • Iterative deepening for time-limited moves
  • Parallel search for higher difficulty levels

Development Challenges & Solutions

Complex Move Validation
Implemented a recursive algorithm to handle multiple jumps and forced capture sequences, ensuring all possible capture chains are evaluated.
AI Performance Optimization
Used memoization and move ordering to reduce search space, achieving 6-ply search depth in under 2 seconds on average hardware.
State Management
Designed immutable game state objects to enable easy undo/redo functionality and game replay system.
GUI Responsiveness
Implemented multi-threading to keep UI responsive during AI calculation, with progress indicators for long computations.
